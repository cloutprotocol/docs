### ink-plugin-reddit

Monitor and interact with Reddit.

```bash
npm install ink-plugin-reddit
```

```typescript
import { redditPlugin } from "ink-plugin-reddit";

const agent = new Agent({
  plugins: [
    redditPlugin({
      clientId: process.env.REDDIT_CLIENT_ID,
      clientSecret: process.env.REDDIT_CLIENT_SECRET,
      username: process.env.REDDIT_USERNAME,
      password: process.env.REDDIT_PASSWORD,

      subreddits: ["cryptocurrency", "ethfinance"],
      postEnabled: false,  // Read-only by default
    }),
  ],
});
```

**Author:** @crypto-dev
**GitHub:** github.com/crypto-dev/ink-plugin-reddit

### ink-plugin-notion

Sync with Notion databases.

```bash
npm install ink-plugin-notion
```

```typescript
import { notionPlugin } from "ink-plugin-notion";

const agent = new Agent({
  plugins: [
    notionPlugin({
      token: process.env.NOTION_TOKEN,
      databases: {
        knowledge: "database-id-1",
        logs: "database-id-2",
      },
      syncInterval: 60000,  // 1 minute
    }),
  ],
});
```

**Author:** @productivity-hacker
**GitHub:** github.com/productivity-hacker/ink-plugin-notion

### ink-plugin-voice

Text-to-speech and voice output.

```bash
npm install ink-plugin-voice
```

```typescript
import { voicePlugin } from "ink-plugin-voice";

const agent = new Agent({
  plugins: [
    voicePlugin({
      provider: "elevenlabs",
      apiKey: process.env.ELEVENLABS_API_KEY,
      voiceId: "your-voice-id",

      // Auto-speak brilliant thoughts
      speakBrilliantThoughts: true,
    }),
  ],
});
```

**Author:** @audio-ai
**GitHub:** github.com/audio-ai/ink-plugin-voice

### ink-plugin-email

Send and receive emails.

```bash
npm install ink-plugin-email
```

```typescript
import { emailPlugin } from "ink-plugin-email";

const agent = new Agent({
  plugins: [
    emailPlugin({
      smtp: {
        host: "smtp.gmail.com",
        port: 587,
        user: process.env.EMAIL_USER,
        pass: process.env.EMAIL_PASS,
      },
      imap: {
        host: "imap.gmail.com",
        port: 993,
      },
      allowedSenders: ["boss@company.com"],
    }),
  ],
});
```

**Author:** @comms-dev
**GitHub:** github.com/comms-dev/ink-plugin-email

## Evaluating Plugins

Before using a community plugin:

<Steps>
  <Step title="Check maintenance">
    - When was the last commit?
    - Are issues being addressed?
    - Is it compatible with latest INK?
  </Step>

  <Step title="Review code">
    - Look for obvious security issues
    - Check how credentials are handled
    - Review external API calls
  </Step>

  <Step title="Test in isolation">
    - Run in development first
    - Monitor API usage
    - Check for rate limit compliance
  </Step>

  <Step title="Check community feedback">
    - Read GitHub issues
    - Search Discord for mentions
    - Look for usage examples
  </Step>
</Steps>

## Contributing Plugins

### Submission Process

1. Build and test your plugin
2. Publish to NPM with `ink-plugin-` prefix
3. Add `ink-agent-plugin` topic to GitHub repo
4. Submit PR to plugins registry

### Requirements

- MIT or compatible license
- TypeScript types included
- README with usage examples
- Passing tests
- No hardcoded credentials

### Plugin Template

```typescript
// Use our template to get started
npx create-ink-plugin my-plugin
```

This scaffolds:
- TypeScript configuration
- Test setup
- Documentation template
- CI/CD workflow
- NPM publishing config

## Security Notes

<Warning>
  Community plugins are not audited. Always review code before use, especially plugins that handle credentials or make financial transactions.
</Warning>

### Safe Practices

1. **Review source code** before installing
2. **Lock versions** in package.json
3. **Monitor network activity** in production
4. **Use separate API keys** for testing
5. **Enable audit logging** for sensitive plugins
