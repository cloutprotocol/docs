Plugins are the primary extension mechanism for ClarkOS agents. They hook into the tick lifecycle, expose actions, and add capabilities without modifying core logic.

## What Plugins Can Do

| Category | Examples |
|----------|----------|
| External integrations | Twitter, Telegram, trading platforms |
| Data sources | News feeds, market data, social mentions |
| Actions | Post content, execute trades, send notifications |
| Analytics | Track metrics, log events, monitoring |

## Plugin Interface

| Field | Type | Description |
|-------|------|-------------|
| `name` | string | Unique identifier |
| `version` | string | Semver version |
| `init` | function | Called on registration |
| `cleanup` | function | Called on agent stop |
| `onTick` | function | Called after each tick |
| `actions` | object | Callable methods |

**Reference:** `Plugin` interface in `src/plugins/types.ts`

## TickContext

Plugins receive context on each tick:

| Field | Type | Description |
|-------|------|-------------|
| `state` | AgentState | Mood, health, routine |
| `memories` | Memory[] | Retrieved memories |
| `knowledge` | Knowledge[] | Retrieved knowledge |
| `recentLogs` | Log[] | Recent activity |

## Using a Plugin

```typescript
import { Agent } from "./src";
import { createTwitterPlugin } from "./plugins/twitter";

const agent = new Agent({
  backend,
  plugins: [
    createTwitterPlugin({
      apiKey: process.env.TWITTER_API_KEY!,
      cooldownMinutes: 60
    })
  ]
});
```

## Building a Plugin

```typescript
export const myPlugin: Plugin = {
  name: "my-plugin",
  version: "1.0.0",
  onTick(context) {
    console.log(`Mood: ${context.state.mood}`);
  }
};
```

**Reference:** See `plugins/` directory for implementation patterns.

**Reference:** See [Example Plugins](/plugins/official) for implementation details.

## Best Practices

- **One responsibility** — Keep plugins focused on a single capability
- **Handle errors** — Catch exceptions to avoid crashing the agent
- **Respect rate limits** — Implement cooldowns for external APIs
- **Test in isolation** — Use `MemoryBackend` for unit testing
